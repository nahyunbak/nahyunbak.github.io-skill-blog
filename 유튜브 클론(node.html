**GET은 어떠한 자원(URL)을 단순히 받기 위해서 사용

  1. req 와 res
req = POST 로 전달한 정보를 요청해서, 정보를 받음 !

res = 정보를 응답! ( res.send )



  1. 총정리

GET request => URL에 접근
POST request => 정보를 전달
- URL을 쓴다(youtube.com) => 브라우저가 GET method를 실행한다 => 브라우저가 URL의 페이지를 읽어온다
- 웹사이트에 Login 한다 => 브라우저가 POST method 실행한다. => 브라우저가 웹사이트에 정보를 전달한다.

ex) 댓글 단다 => POST request 만듬(댓글 정보전달) => 해당 request 서버로 전달 => 서버는 실행한다.(어떤방식으로든.. 하지만 이건 저장한다는 뜻은 아님)

req obj : 누군가 정보를 얻고자 페이지를 요청했을때, 어떤 종류의 데이터가 페이지로 전송됐는지 확인할때 request object를 이용한다. 예를 들어 이 URL에 POST 방식으로 아이디와 패스워드를 전송한다면, 이렇게 app.post로 아이디랑 패스워드를 보낸다면 서버에는 request object로 정보를 얻을 수 있다.

res obj => POST한 사람의 정보를 받아서 보내주는 역할을 한다.

기본적 작동방식 : 서버 생성하고 route를 생성하고 그것에 응답하는 방식==> express와 node.js로 간단히 할 수 있음

//미들웨어 추가 : 중간에 있는 놈들은 늘 세 번째 변수, next를 붙여준다. 그리고 함수 안에 next()를 붙여준다.

const betweenHome = (req, res, next) => {
    console.log("I'm between");
    next()
}

app.use(betweenHome);
